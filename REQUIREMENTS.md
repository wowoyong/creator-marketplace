# 크리에이터 마켓플레이스 플랫폼 - 요구사항 정의서

## 📋 프로젝트 개요

### 프로젝트명
**크리에이터 마켓플레이스** (가칭)

### 목적
작가(크리에이터)와 클라이언트를 연결하는 커미션 기반 마켓플레이스 플랫폼 구축

### 참고 플랫폼
- crepe.cm (한국 작가 커미션 플랫폼)
- 당근마켓 (1:1 채팅 기반 거래 방식)
- Fiverr (글로벌 프리랜서 마켓플레이스)

---

## 🎯 비즈니스 모델

### MVP 단계 (Phase 1)
- **수익화 없음** - 사용자 확보 및 검증에 집중
- 작가-클라이언트 간 직접 거래 (플랫폼 외부 결제)
- 플랫폼은 거래 상태만 추적 (거래중/완료)

### 향후 전환 계획 (Phase 2+)
- **플랫폼 에스크로 시스템** 도입
- 토스페이먼츠 등 PG 연동
- 거래 수수료 부과 (예: 10-15%)
- DB 스키마는 에스크로 전환을 고려하여 설계

---

## 👥 사용자 유형

### 1. 작가 (Artist/Creator)
- 포트폴리오 등록 및 관리
- 의뢰 요청 수신 및 응답
- 작업물 납품
- 후기 관리

### 2. 클라이언트 (Client/Buyer)
- 작가 검색 및 탐색
- 의뢰 요청 발송
- 작업 진행 상황 확인
- 후기 작성

## 🎨 핵심 기능 (MVP 필수)

### 1. 회원 관리
- **로그인**: 카카오 OAuth만 지원
- **회원 유형**: 가입 시 작가/클라이언트 선택
- **프로필**: 기본 정보 + 역할별 추가 정보

### 2. 작가 프로필 & 포트폴리오
- **필수 요구사항**:
  - 포트폴리오 이미지 **최소 5장** 업로드
  - 참고 URL 추가 가능 (선택)
  - 작가 소개, 전문 분야, 가격대 등
- **이미지 저장**:
  - MVP: Mac Mini 서버 로컬 저장 (`/uploads`)
  - 향후: Cloudflare R2 또는 AWS S3로 마이그레이션
  - DB에는 전체 URL 저장 (마이그레이션 용이성)

### 3. 작가 검색 & 탐색
- 작가 목록 페이지
- 기본 검색 (이름, 키워드)
- 작가 상세 페이지 (포트폴리오 갤러리)

### 4. 1:1 채팅 시스템
- **실시간 채팅**: Socket.IO 사용 (Someone 프로젝트 재사용)
- 작가-클라이언트 간 직접 소통
- 의뢰 내용 협의 및 가격 조율
- 파일 첨부 기능 (참고 이미지 등)

### 5. 거래 상태 관리
- **거래 플로우**:
  1. **의뢰 요청** - 클라이언트가 작가에게 요청
  2. **거래중** - 작가 수락 후 작업 진행
  3. **거래완료** - 작업 완료 및 납품
  4. **평가** - 양측 후기 작성
- **결제**: 플랫폼 외부에서 처리 (계좌이체, 토스 등)
- **상태 표시**: 플랫폼에서 진행 상황만 추적

### 6. 후기 시스템
- 거래 완료 후 양측 평가 가능
- 별점 (1-5) + 텍스트 후기
- 작가 프로필에 후기 목록 표시
- 평균 평점 계산

### 7. 알림 시스템 (3채널)
- **인앱 알림**: 플랫폼 내 알림 목록 + 배지
- **이메일 알림**: 중요 이벤트 (의뢰 요청, 거래 완료 등)
- **푸시 알림**: PWA 브라우저 푸시
- **알림 이벤트**:
  - 새 채팅 메시지
  - 의뢰 요청/수락
  - 거래 상태 변경
  - 후기 작성
- **사용자 설정**: 채널별 on/off 가능

## 🛠 기술 스택

### 프론트엔드
- **프레임워크**: Next.js 15 (App Router)
- **언어**: TypeScript
- **스타일링**: TailwindCSS (또는 기존 스타일 시스템)
- **상태관리**: React Context / Zustand
- **PWA**: Service Worker + Web Push API

### 백엔드
- **프레임워크**: NestJS 11
- **언어**: TypeScript
- **데이터베이스**: PostgreSQL
- **ORM**: Prisma
- **실시간 통신**: Socket.IO
- **인증**: Passport.js + Kakao OAuth

### 인프라
- **서버**: Mac Mini (기존 Someone 프로젝트와 동일)
- **프로세스 관리**: PM2
- **파일 저장**: 로컬 저장 (향후 Cloudflare R2)
- **이메일**: SendGrid 또는 AWS SES
- **모니터링**: (기존 Someone 설정 재사용)

### 개발 도구
- **버전 관리**: Git + GitHub
- **코드 품질**: ESLint + Prettier
- **테스트**: Jest + Supertest (백엔드), Vitest (프론트엔드)
